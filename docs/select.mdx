`<Select>`는 native `<select>` 기반입니다. shadcn식 커스텀 dropdown을 거부 — **native가 접근성·모바일 OS·키보드 모두 지원**.

```tsx
import { Select } from "@freeive/anti-card";

<Select
  label="카테고리"
  placeholder="선택하세요"
  options={[
    { value: "learning", label: "학습 일지" },
    { value: "ai", label: "AI 워크플로우" },
  ]}
/>
```

## Props

| Prop | Type | Description |
|---|---|---|
| `label / error / hint` | `ReactNode` | Input과 동일 |
| `options` | `SelectOption[]` | `{value, label, disabled?}[]` |
| `placeholder` | `string` | 빈 값 옵션 (selected="" 일 때) |
| `wrapperClassName` | `string` | wrapper className |
| `...rest` | `SelectHTMLAttributes<HTMLSelectElement>` | value / onChange / required 등 |

## 디자인

- Input과 동일 톤 (rounded-md, border, focus emerald)
- `appearance-none` + 커스텀 SVG 화살표 (12x8px, zinc-500)
- 우측 padding 8 (화살표 자리)

## native vs custom dropdown

| | native | custom |
|---|---|---|
| 접근성 | ✅ 자동 | 직접 구현 필요 |
| 모바일 OS UI | ✅ 자동 (iOS picker, Android dialog) | ❌ |
| 키보드 (방향키, type-ahead) | ✅ 자동 | 직접 구현 |
| 시각 커스텀 | ⚠️ 제한적 | ✅ 자유 |

대부분의 경우 native가 충분 — 시각 커스텀이 정말 필요한 경우만 별도 구현.

## children 사용

`options` 대신 `<option>` 직접 사용 가능 (optgroup 등):

```tsx
<Select label="카테고리">
  <optgroup label="학습">
    <option value="learning">학습 일지</option>
    <option value="review">회고</option>
  </optgroup>
</Select>
```
