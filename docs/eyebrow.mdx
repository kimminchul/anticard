`<Eyebrow>`는 섹션 위에 붙는 작은 카테고리 라벨입니다. 박스나 카드 없이도 영역을
구분하는 안티 카드 5원칙 중 하나(smallcaps 라벨).

```tsx
import { Eyebrow } from "@freeive/anti-card";

<Eyebrow>Framework · v0</Eyebrow>
<h1>제목</h1>
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `tone` | `"neutral" \| "accent"` | `"neutral"` | `accent`는 액센트 컬러로 강조 |
| `className` | `string` | — | Tailwind 클래스 추가 (twMerge로 충돌 해결) |
| `...rest` | `HTMLAttributes<HTMLParagraphElement>` | — | 표준 p 속성 |

## 톤 변형

```tsx
<Eyebrow>Heritage · 2016 — Now</Eyebrow>          // neutral (기본)
<Eyebrow tone="accent">Live · in progress</Eyebrow>  // accent
```

## 디자인 메모

- 글자 크기 12px, letter-spacing 0.08em, 대문자 변환
- `font-medium`으로 너무 가늘지 않게
- `data-anti-card="eyebrow"` attribute로 외부 스타일 타겟팅 가능

## 자주 묻는 패턴

### 헤딩 위 라벨

```tsx
<section>
  <Eyebrow>Pillars</Eyebrow>
  <h2 className="mt-3 text-3xl font-semibold tracking-tight">
    네 개의 축으로 운영합니다.
  </h2>
</section>
```

### 색조 변경 (Tailwind class)

```tsx
<Eyebrow className="text-yellow-500">env not set</Eyebrow>
```

`tone`은 사전 정의 두 가지뿐. 더 다양한 색은 `className` 사용을 권장합니다.

## HTML 레퍼런스 (framework-free)

React 안 쓰는 환경(Vue·Svelte·순수 HTML)에서 같은 톤을 만들고 싶을 때 그대로 복사:

```html
<!-- neutral -->
<p class="text-[12px] uppercase tracking-[0.08em] text-zinc-500 font-medium">
  Heritage · 2016 — Now
</p>

<!-- accent -->
<p class="text-[12px] uppercase tracking-[0.08em] text-emerald-400 font-medium">
  Live · 진행 중
</p>
```

이 패턴이 안티 카드의 진짜 정의입니다. React 컴포넌트 `<Eyebrow>`는 이 클래스
조합에 변형 옵션과 className 머지를 더한 편의 wrapper일 뿐입니다.
