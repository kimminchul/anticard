> 시장에 없던 것은 컴포넌트가 아니라 AI 디자인 가이드입니다.

## 두 레이어

```
Layer 1 — AI Skill   (진짜 본체)
└── Claude · Cursor 에 설치 → AI가 화면을 만들 때 참조하는
    원칙·패턴·HTML 레퍼런스
    
Layer 2 — React 컴포넌트   (보조)
└── npm install @freeive/anti-card
    Eyebrow / SectionFrame / ListRow / …
```

대부분의 UI 라이브러리는 Layer 2 만 제공합니다. 안티 카드는 Layer 1이 본체.

## 왜 Skill인가 — 문제 정의

ChatGPT · Claude · Cursor · v0 가 화면을 만들 때 거의 다음과 같이 합니다:

1. shadcn/ui의 `Card` 컴포넌트로 시작
2. 카드 안에 `Header`, `Content`, `Footer` 카드를 또 넣음
3. 그리드 3컬럼으로 나열
4. Tailwind dark 모드 클래스 추가

→ 결과적으로 **모든 AI 생성 화면이 비슷해집니다.**

## 해결 — 디자인 결정 기준을 주입

AI Skill은 AI에게 다른 결정 기준을 줍니다:

- "카드를 쓰지 마세요. 위계는 헤어라인 + 큰 헤딩 + smallcaps 라벨로."
- "그리드 대신 행 레이아웃 (`divide-y border-y`)을 우선 검토."
- "섹션 사이 여백은 100~160px. 빽빽하게 채우지 말 것."
- "헤딩은 `text-[clamp(2rem,5vw,4rem)] font-semibold tracking-tight`."

이런 원칙이 AI의 system prompt 또는 cursorrules에 들어가면, AI가 만든
화면이 즉시 안티 카드 톤을 띱니다.

## Framework-agnostic

React 컴포넌트(Layer 2)와 달리 Skill은 어떤 framework로 만들든 동작합니다.

- React 프로젝트? → Tailwind 클래스로 안티 카드 톤
- Vue 프로젝트? → 같은 클래스, 같은 톤
- 순수 HTML? → 그대로 적용 가능
- Svelte? Astro? → 동일

→ 컴포넌트가 없어도 Skill만으로 안티 카드 디자인 적용.

## 설치 (예정)

> 0.0.x 단계에서는 manual 셋업. 0.1.0에서 정식 install 명령 제공.

### Claude Code

```bash
# (예정) 한 줄 설치
claude skill install @freeive/anti-card
```

지금은 [skill/CLAUDE.md](https://github.com/kimminchul/anticard/blob/main/skill/CLAUDE.md) 파일을
프로젝트 루트의 CLAUDE.md 에 복사 붙여넣기.

### Cursor

`.cursorrules` 파일에 [skill/cursorrules](https://github.com/kimminchul/anticard/blob/main/skill/cursorrules) 내용
붙여넣기.

### 다른 AI 도구

skill/ 디렉토리의 system prompt 파일들을 그대로 사용.

## 어떻게 작동하나 — 예시

**프롬프트**: "Heritage 페이지에 50개 클라이언트 리스트 만들어줘"

**Skill 없을 때 (전형적인 AI 출력)**:
```tsx
<div className="grid grid-cols-3 gap-4">
  {clients.map(c => (
    <Card>
      <CardHeader>
        <CardTitle>{c.name}</CardTitle>
      </CardHeader>
      <CardContent>{c.year}</CardContent>
    </Card>
  ))}
</div>
```

**Skill 있을 때 (안티 카드 톤)**:
```tsx
<section className="py-16 md:py-20">
  <p className="text-[12px] uppercase tracking-[0.08em] text-zinc-500">Sectors</p>
  <h2 className="mt-3 text-2xl font-semibold tracking-tight">
    어떤 무대에서 일했는가.
  </h2>
  <ul className="mt-12 divide-y divide-white/[0.06] border-y border-white/[0.06]">
    {clients.map(c => (
      <li className="grid grid-cols-[140px_1fr_auto] py-6 gap-8">
        <span className="text-[12px] uppercase text-zinc-500">{c.year}</span>
        <span className="text-[15.5px]">{c.name}</span>
        <span className="text-[12.5px] text-zinc-400">{c.industry}</span>
      </li>
    ))}
  </ul>
</section>
```

→ 같은 정보, 다른 디자인 결정. **AI가 만든 것 같지 않은 화면.**

## Skill의 구성 요소 (앞으로)

- 안티 카드 5원칙 본문
- 카드를 안 쓰고 위계 표현하는 6가지 패턴
- 좋은 예 / 나쁜 예 (paired examples)
- HTML 레퍼런스 (framework-free)
- 자주 만드는 페이지 패턴 (Hero / Sectors / Pricing / FAQ / Footer)
