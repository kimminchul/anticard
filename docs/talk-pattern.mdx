`<TalkPattern>`은 작은 랩의 **"잘 맞는 의뢰만 함께"** 정책을 표현하는 Talk·Contact 페이지 패턴입니다. 일반 폼 페이지 거부 — 잘 맞는 일 / 다른 분께 더 잘 맞을 일을 미리 명시.

```tsx
import { TalkPattern } from "@freeive/anti-card";

<TalkPattern
  eyebrow="Talk · 함께 일하기 전에"
  title="어떤 의뢰가 잘 맞을까요."
  lead="시간이 한정되어 있어, 양쪽 모두 좋은 결과를 만들 수 있는 의뢰만 받습니다."
  acceptList={[
    "의사결정자가 직접 대화에 함께하는 의뢰",
    "왜 그렇게 만드는지 함께 고민할 수 있는 의뢰",
    "인터랙션과 UX를 중요하게 생각하는 의뢰",
  ]}
  declineList={[
    "정해진 템플릿대로 빠르게 마무리하는 단순 홈페이지",
    "가격이 가장 큰 기준인 의뢰",
    "방향·의도보다 지시 이행이 우선인 의뢰",
  ]}
  channels={[
    { label: "Email", value: "ive@freeive.com", href: "mailto:ive@freeive.com" },
    { label: "GitHub", value: "kimminchul/anticard", href: "https://github.com/kimminchul/anticard", external: true },
  ]}
/>
```

## Props

| Prop | Type | Description |
|---|---|---|
| `eyebrow / title / lead` | `ReactNode` | Hero 영역 (HeroPattern 내장) |
| `acceptList` | `ReactNode[]` | 받는 일 (✓ emerald) |
| `declineList` | `ReactNode[]` | 안 받는 일 (✕ zinc) |
| `channels` | `Array<{label, value, href?, external?}>` | 연락 채널 (DefList 패턴) |

## 정체성

받는 일을 명시하는 것보다 **잘 맞지 않는 일을 미리 명시하는 것이 양쪽 시간을 아껴주는 시그널**입니다.
"이런 일은 다른 분께 더 잘 맞아요"를 부드럽게 전하면 의사결정자가 자신의 의뢰가 적합한지 빠르게 판단할 수 있습니다.
