`<Grid>`는 단순 그리드 레이아웃 wrapper입니다. 모바일 → 데스크톱 컬럼 자동 전환.

```tsx
import { Grid } from "@freeive/anti-card";

<Grid columns={3}>
  <Image src="/1.jpg" />
  <Image src="/2.jpg" />
  <Image src="/3.jpg" />
</Grid>
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `columns` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | `3` | 데스크톱 컬럼 |
| `mobileColumns` | `1 \| 2` | `1` | 모바일 컬럼 |
| `gap` | `"tight" \| "default" \| "loose"` | `"default"` | 간격 |
| `as` | `"div" \| "ul" \| "section"` | `"div"` | 시맨틱 |

## gap

| gap | 값 |
|---|---|
| `tight` | gap-3 |
| `default` | gap-6 md:gap-8 |
| `loose` | gap-10 md:gap-14 |

## 카드 그리드와의 차이

`<Grid>`는 **정렬**만 제공 — 박스 / border / bg / shadow 없음. "카드 그리드"는 안티 카드 정체성에 위배.
정보 나열은 `<ListRow>` 또는 `<ul>` + 헤어라인 권장. Grid는 **이미지 / FeatureRow 등 본질이 grid**인 경우만.

## Grid vs GridSystem

| | Grid | GridSystem |
|---|---|---|
| 분할 | 균등만 | 자유 (3+9, 8+4 등) |
| 용도 | 이미지·작은 카드 균등 | 사이드바+본문, asymmetric |

자유 layout이 필요하면 [GridSystem](/anti-card/grid-system) 사용.
