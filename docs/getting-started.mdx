`@freeive/anti-card`는 [Freeive](https://freeive.com)가 **AI 코딩 도구의
디자인 동질화 문제**에 답하기 위해 만드는 React + Tailwind UI 시드입니다.

> **이 문서는 패키지 안에 함께 배포됩니다.** `node_modules/@freeive/anti-card/docs/`
> 에서 직접 읽거나, [freeive.com/anti-card](https://freeive.com/anti-card)
> 에서 동일한 내용을 볼 수 있습니다.

## 왜 카드가 아닌가

ChatGPT · Claude · Cursor · v0 — AI 도구가 생성하는 화면이 거의 다 같은 모양으로
수렴합니다. 대부분 **카드 안에 카드를 넣어** 위계를 표현합니다. 시각적으로 안전하지만,
모든 사이트가 같은 모양이 되는 비용이 큽니다.

위계는 카드의 그림자가 아니라 **공간·대비·여백**으로도 만들 수 있습니다.
안티 카드는 이쪽으로 다시 짜는 시도입니다. 자세한 원칙은
[왜 카드가 아닌가](/anti-card/why-not-cards) 참고.

## 들어갈 항목

- 디자인 원칙 — 카드를 쓰지 않고 위계를 만드는 다섯 가지 패턴
- 컴포넌트 — Tailwind 기반 React 컴포넌트
- AI 스킬 — Claude / Cursor 가 안티 카드 톤으로 만들도록 한 줄로 가이드
- 활용사례 — 모청, [freeive.com](https://freeive.com) 자체

## 설치

NPM 발행은 0.1.0부터입니다. 그 전까지 로컬에서 검증하려면:

```bash
# 옵션 A — npm link (권장, 매번 publish 불필요)
cd anti-card    && npm link
cd your-app     && npm link @freeive/anti-card

# 옵션 B — tarball (Turbopack 호환 가장 안전)
cd anti-card    && npm pack
cd your-app     && npm install ../anti-card/freeive-anti-card-0.0.1.tgz
```

0.1.0 이후:

```bash
npm install @freeive/anti-card
```

## 빠른 사용

```tsx
import { SectionFrame, ListRow, Eyebrow } from "@freeive/anti-card";

export default function Heritage() {
  return (
    <SectionFrame
      eyebrow="Heritage · 2016 — Now"
      title="큰 프로젝트들의 깊이를 랩으로 옮긴다."
      description="텔레콤·금융·교육의 큰 싸움을 거친 깊이가 토대입니다."
    >
      <ul className="divide-y divide-white/[0.04] border-y border-white/[0.04]">
        <ListRow meta="2024" trailing="EBS">EBS 온라인 클래스 재구조화</ListRow>
        <ListRow meta="2023" trailing="롯데카드">mydata 수집 및 admin 개발</ListRow>
      </ul>
    </SectionFrame>
  );
}
```
